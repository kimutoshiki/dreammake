# 00. プロジェクト概要

## 🎯 ビジョン

**「AIに出てこないのは誰か」を問い続ける、小学校社会科の探究プラットフォーム。**

小学生が、調べる → まとめる → AI に教える → 作品にする → 共有する → さらに深める、の探究サイクルを通じて、**自分の想定に収まらない立場**に触れたとき、すぐに退けたり同調したりせず考え続けようとする構え(**他者性**)を育てる。

単なる AI チャットアプリではない。**生成 AI が多数派の声を返しやすい性質そのものを教材化**し、子どもが「AI に出てこないのは誰か」を問うことで、立場選定の基準を自分で吟味する学びを成立させる、教育研究と連動した授業プラットフォームである。

---

## 💡 体験のコア

```
調べる → まとめる → AI に教える → 作品にする → 共有する → さらに深める
  ↑          ↓         ↓                              │
  └── 「AIに出てこないのは誰?」を繰り返し問い直す ──┘
              立ち止まりの言葉を自分で可視化
              自分の立場を時系列で記録する
```

- **調べる**: 本・図鑑・取材・Web・身の回りの観察から一次情報を集める
- **まとめる**: 集めた情報を Q&A カード / 絵 / 図にして整理(出典必須)
- **AI に教える**: 自分のボットに「これを知ってるよ」と登録 → 対話でテスト
- **作品にする**: インフォグラフィック / 画像 / 動画 / 音楽 / クイズ / 疑似アプリ
- **共有する**: クラス内の「しらべもの広場」で対話、リミックス、立場マップで少数派を見える化
- **さらに深める**: 「AI に出てこないのは誰?」 → 他者性への感受性を耕す
- **自分を見つめる**: 立ち止まりの言葉(でも/なぜ/別の見方をすれば)を自分で確かめる

---

## 🧭 教育研究と設計原理

本アプリは、**生成AI時代の社会科授業原理**の検証を組み込んで設計されている。

### 踏まえる研究系譜
- 学習指導要領解説 社会編・文部科学省 生成AI暫定ガイドライン: 多面的・多角的考察と批判的活用を要請
- 池野範男らの論争問題学習・公共的判断研究: 多様な立場の提示が判断の質を高める
- **これまで扱われてこなかった**: 生成AIが多数派の声を返しやすい性質を教材化し、「AIに出てこないのは誰か」を問いの核に据える実践

### 応答する空白
- AI の限界を**学びの触媒**に転じる授業原理を、検証可能な形で提供
- **AI 挿入の 3 タイミング**を設計:before-self(前)/after-self(後)/ask-missing(欠けた声を問う)
- 立場選定の基準そのものを子どもが吟味する学び
- 量的指標(立ち止まりの言葉の頻度、立場の数、Wilcoxon 検定)と質的指標(エピソード記述、KH Coder 風の共起分析)の両輪
- 授業者が**単元設計** → **事前調査** → **単元実施** → **事後分析**の一連を、アプリ内で完結できる

## 👥 ターゲットユーザー

### 主ユーザー: 小学 3〜6 年生
- ひらがな・カタカナ・学年相当の漢字+ふりがなで操作できる
- 総合的な学習の時間・社会科・自由研究・クラブ活動での利用を想定
- タブレット(iPad / Chromebook)を授業で日常的に使っている

### 副ユーザー: 小学 1〜2 年生
- 音声入力・読み上げ中心、イラスト主体の UI

### 管理者: 教員
- 単元設計、事前事後アンケート作成、児童の立場分布・立ち止まりの言葉の推移を俯瞰
- エピソード記述のレビュー・承認、共起分析の可視化
- 研究倫理ワークフロー(保護者同意、匿名化、データ保管期限、対話ログ返却)

### 連携: 保護者
- 教員経由で同意書を受領・同意を記録
- 児童の対話ログは、教員経由で保護者に返却可(独立した保護者ポータルは持たない)

## 🧪 想定する学習シーン

| シーン | 例(単元テーマと問い) |
|--------|----|
| 社会科(地域) | 「わたしたちの町の昔と今」: この町の未来を決めるとき、だれの声が聞かれていない? |
| 社会科(産業) | 「働く人のはなし」: 便利さは誰のがまんの上にあるか? |
| 社会科(歴史) | 「昔の人のくらし」: 歴史書に名前が残らなかった人は誰か? |
| 社会科(国土) | 「川と森と人」: この川の声を聞くと、私たちはどう暮らすか? |
| 総合的な学習 | 横断的な探究として 10〜15 時間の中単元 |
| 自由研究 | 夏休みの研究をボットと対話しながら深める |

## 🎓 目指す姿

- **教員が検証可能な授業を設計できる**: 授業原理を「具体的な機能と指標」として提供
- **子どもが自分の思考の足跡を見られる**: 立ち止まりの言葉セルフビュー、立場の時系列記録
- **AIの限界を子ども自身が確かめる**: 「AIに出てこないのは誰?」機能の常設
- **表現と思考が結びつく**: 作品づくりは常に「誰に何を伝えるか」を持つ
- **教科横断が自然に起きる**: 社会を主軸に、国語・図工・音楽・総合・プログラミングと往復

---

## ❌ 非目標(このプロダクトがやらないこと)

1. **汎用 AI チャット**: ChatGPT 的な何でも聞けるアシスタントは目的ではない
2. **宿題の答えを出す**: 問題を解くためのツールではない。**プロセス**を支援する
3. **児童間の自由なメッセージング**: SNS 的な機能は持たない
4. **児童の個人を特定する情報の収集**: 本名・住所・電話・学校名は要求しない
5. **保護者の代替**: 子どもの悩み相談の受け皿ではない。危険な兆候は即エスカレート
6. **教員の代替**: AI は触媒。授業者の判断・価値観・関係性は置き換えられない

---

## 🧭 プロダクト原則

1. **子どもの安全 > すべて**
2. **他者性の育成 > AI の便利さ**: AI が先回りして立場を確定しない
3. **出典の尊重 > 滑らかな応答**: 「調べていないこと」は堂々と「調べていない」と言う
4. **「AIに出てこないのは誰?」を忘れない**: 全機能でこの視座を失わない
5. **全学年で動く > 特定学年に最適化**
6. **教員の負担軽減 > 高機能**
7. **ローカルで動く > クラウド便利機能**: 学校 Wi-Fi 不安定でも主要機能は動く
8. **研究と実践は同じ基盤で**: 授業実践が研究データになり、研究知見が次の授業を更新する

---

## 📏 成功の測り方(将来の KPI 例)

- **探究サイクル完走率**(ボット作成 → 対話 → 共有 → AIに出てこないのは誰? → 再探究)
- **1 単元あたりの立場の多様性**(`StanceSnapshot` のユニーク立場数)
- **立ち止まりの言葉の事前事後変化**(ウィルコクソン検定の p 値と実質効果量)
- **「AIに出てこないのは誰?」機能の活用回数**(1 児童 1 単元あたり)
- **出典ありナレッジの比率**
- **教員が単元設計 → 分析 → 報告まで完結できるか**(投入時間)
- モデレーションのハードブロック率(低いほど健全な利用)
