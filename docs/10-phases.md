| Phase | テーマ | 主な成果物 | 推定期間目安 |
|-------|--------|-----------|-------------|

## 🚀 Phase 1 — コア MVP

### 目的
「認証 → マイチャットボット作成 → ナレッジ登録 → 対話 → クラス共有」までの**一本の導線**を、エンドツーエンドで動かす。安全性は妥協しない。

### スコープ
#### 含む
- 認証(児童=絵柄、教員=マジックリンク)
- 保護者同意の**教員経由取得**(アプリ内で、教員が児童ごとに同意状況を管理)
- ボット作成(3ステップ)、ナレッジ登録(Q&A / ノート)、出典入力必須化
- Claude Sonnet による対話(SSE ストリーミング、出典自動付与、他者性の種の埋め込み)
- 入力モデレーション(ルール + Haiku)、出力モデレーション(Haiku)
- クラス内広場、リミックス(出典継承)
- 学年プロファイル(lower/middle/upper)、ふりがな、音声
- 教員ダッシュボードの**最小版**
- SQLite + Prisma、Local FS ストレージ、docker compose

#### 含まない
- Unit モデル(Phase 2)
- 画像・動画・インフォグラフィック・クイズ(Phase 3)
- 研究機能(Phase 2-4 で段階的)
- つくってみようモード(Phase 5)

### 完成定義(抜粋)
- [ ] 児童が 10 秒以内にログインできる
- [ ] ボット作成 → ナレッジ 3 件登録 → 対話 → クラス公開 が一気通貫で可能
- [ ] ナレッジ外の質問に「まだ調べていないよ」が返る
- [ ] すべての AI 応答に出典が付く
- [ ] ハードブロック入力が児童に返らない
- [ ] 3 帯プロファイル差分が画面に反映される
- [ ] Lighthouse Perf 85+ / A11y 100

## 🔬 Phase 2 — 探究単元と他者性の道具立て

### 目的
実習Ⅲの授業原理(**AIに出てこないのは誰?**、3 つの AI 挿入タイミング、立ち止まりの言葉の追跡)を、**すべての児童・すべての単元で使える常設機能**として実装する。

### スコープ
#### 含む
- **Unit / UnitHour** モデルの実装(教員が単元を設計)
- 単元トップ画面(17)、立場マップ(18)
- **「AIに出てこないのは誰?」画面(19)**: [missing-voice-probe.md](04-prompts/missing-voice-probe.md)
- **立ち止まりの言葉セルフビュー(20)**: [standstill-detection.md](04-prompts/standstill-detection.md)(児童向け自己確認)
- `StanceSnapshot`, `ReflectionEntry`, `MissingVoiceHypothesis` の記録
- **事前事後アンケート**(画面 13 の設問作成、児童回答、基本集計)
- 教員の単元設計 UI(画面 13)の第一版
- 対話ログの児童本人への返却(テキスト出力)
- 教員ダッシュボードに単元タブ追加(進捗・回答率・簡易サマリ)

#### 含まない
- AI によるエピソード抽出(Phase 4)
- 共起分析の可視化(Phase 4)
- Wilcoxon 符号順位検定(Phase 4)
- 単元骨子の AI 自動提案(Phase 4)

### 完成定義
- [ ] 教員が 30 分で 1 単元を設計・公開できる
- [ ] 児童が単元内で対話 → 立場記録 → AI の偏り質問 → 振り返り の流れを実行できる
- [ ] ルールベース + Haiku で立ち止まりの言葉が記述から検出される
- [ ] pre/post アンケートが配布・集計できる(統計処理なし、分布表示まで)
- [ ] 対話ログを児童本人がテキストで出力できる
- [ ] Unit.researchMode=false の単元では、研究用の追加データ収集が走らない
- [ ] すべての単元で「AIに出てこないのは誰?」ボタンが CTA として機能する

### 前提・依存
- Phase 1 の対話基盤
- kuromoji.js の辞書を public に配置

### 対処するリスク
R-22(AI の偏りに気づけない児童)、R-23(研究データの扱い)、R-01 の強化

## 🎨 Phase 3 — 表現(画像・インフォ・動画・音楽・クイズ)

### 目的
ナレッジや立場を**視覚・聴覚・体験に変える**表現機会を提供。
図工・音楽・国語との教科横断の要。

### スコープ
#### 含む
- **画像つくろう**(画面 8): Gemini 連携、プロンプトコーチ、安全化フィルタ
- **インフォグラフィックつくろう**(画面 9): 4 テンプレート、編集エディタ
- **動画つくろう**(画面 21): スライド+TTS+BGM の単一 MP4 合成(Remotion for Browser)
- **音楽** (`Artwork.kind='music'`): Tone.js ベースの簡易作曲、mood ラベルから生成
- **クイズ・少数派さがし**(画面 22): `Artwork.kind='quiz'`、`quizKind='stance-match'|'minority-find'`
- 画像・PDF の OCR(Claude Vision)
- `ImageGenAdapter`(Gemini + OpenAI フォールバック)、`VideoAdapter`(ブラウザ内合成)

#### 含まない
- 動画の長尺(MVP は最大 60 秒)
- 外部動画共有

### 完成定義
- [ ] 対話 4〜5 ターンで画像生成が完結
- [ ] インフォグラフィックが 4 テンプレで生成・編集、出典維持
- [ ] 60 秒以内の動画(3〜5 スライド+ナレーション+BGM)を児童が作成、ローカル再生可能
- [ ] 簡易作曲で mood 別の 30 秒 BGM が生成
- [ ] クイズが `StanceSnapshot` を生データとして少数派さがしを可能にする
- [ ] OCR 結果が編集可能、誤認識を児童が直せる

## 📊 Phase 4 — 教員研究機能 + ポートフォリオ + アクセシビリティ

### 目的
実習Ⅲの質的・量的分析、単元設計の AI 支援、研究アウトプットの下書き作成を完結させる。
加えて、ポートフォリオとアクセシビリティで、全児童・全保護者・全教員に使える完成度へ。

### スコープ
#### 含む
- **単元設計の AI 提案**: [unit-scaffold.md](04-prompts/unit-scaffold.md)、[pre-post-survey-gen.md](04-prompts/pre-post-survey-gen.md)
- **共起分析ビュー**(画面 16): [co-occurrence-summary.md](04-prompts/co-occurrence-summary.md)、kuromoji+独自実装+ Claude 要約
- **エピソード抽出と教員レビュー**(画面 15): [episode-extractor.md](04-prompts/episode-extractor.md)
- **Wilcoxon 符号順位検定**: `lib/research/stats.ts`、画面 14 で pre/post 比較
- **研究倫理ワークフロー**: `Unit.ethicsApproval`、`ConsentRecord(kind='research-participation')`、単元公開前チェックリスト
- **対話ログ PDF 返却**(児童本人向け、保護者にも教員経由で渡せる)
- **研究アウトプット PDF**: 単元サマリ、事前事後結果、共起、エピソード、個別軌跡の書き出し
- **ポートフォリオ機能**: 作品・出典・振り返りの時系列、年度末 PDF
- **ワークシート PDF**: 自由研究テンプレートからの出力(家庭学習との接続、保護者ポータルなし)
- **グループ探究モード**: 複数児童で 1 つのボットを共同編集(変更履歴)
- **アクセシビリティ強化**: 色覚多様性、ハイコントラスト、キーボード完全対応、音声の速度/ピッチ調整、文字サイズ 3 段階

#### 含まない
- ~~英語モード~~(削除済み)
- ~~保護者向け独立ポータル(週次レポート閲覧)~~(削除済み、同意は教員経由)
- 多校・他校種への拡張(他校交流は Phase 5 以降)

### 完成定義
- [ ] 教員が 10 分で単元骨子の AI 提案を受け、編集して公開できる
- [ ] 単元終了後、教員が pre/post の Wilcoxon 結果と cautions を読める
- [ ] エピソード記述候補が AI 生成 → 編集 → 承認されて PDF に含まれる
- [ ] 共起ネットワーク(pre/mid/post)と立ち止まり語の共起パートナー変化が可視化
- [ ] 研究アウトプット PDF(1 単元ごと)が A4 15〜30 枚で書き出せる
- [ ] ポートフォリオが 1 児童 A4 10 枚以内でまとまる
- [ ] WCAG 2.1 AA 準拠、キーボード全機能操作可能

### 前提・依存
- Phase 2 で Unit 系データが蓄積されていること
- Phase 3 の表現作品も含めて portfolio を組めること

## 🧰 Phase 5 — つくってみようモード

### 目的
プログラミング教育との接続。児童が「立場当てクイズ」「意見箱」を自然言語でつくる体験。

### スコープ
#### 含む
- 画面 10(つくってみようモード)
- [mini-app-codegen.md](04-prompts/mini-app-codegen.md) によるコード生成
- 静的スキャン + iframe サンドボックス + CSP(5 層防御、詳細 [05-safety-and-privacy.md](05-safety-and-privacy.md))
- 児童が作ったアプリを単元の `UnitArtwork` として紐付け
- クラス配信(他の児童の端末で動かす簡易共有)

#### 含まない
- 複数ファイル管理
- 外部ライブラリ利用
- サーバーサイド実行

### 完成定義
- [ ] 10 種の代表リクエストで動くアプリが生成される
- [ ] 静的スキャン + サンドボックスで禁止 API が動かない
- [ ] サンドボックス脱出試験 20 件で脱出 0 件
- [ ] 教員がクラス単位で機能 ON/OFF できる

### 前提・依存
- Phase 1 の Sonnet 呼び出し基盤
- 単元モデルに UnitArtwork ができていること(Phase 2)

## 🔁 フェーズ間の原則

### 「前フェーズの安定化」を終えてから次へ
- 主要不具合 0 件、重大リスク許容範囲、教員・保護者・開発で合意

### フェーズ境界での API 契約
- データモデルの破壊的変更を避ける(ADD 優先)
- Adapter インタフェースは前方互換(追加メソッドは optional に)

### フェーズ内の「MVP → 改善」サイクル
- 各フェーズ前半 60% は主要機能の MVP、後半 40% は UX 改善・例外処理・テスト厚み
