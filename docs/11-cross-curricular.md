# 11. 教科等横断的な学びの設計

> 社会科を**主軸**に、国語・図工・音楽・総合的な学習の時間・プログラミング教育を
> **自然に接続**する原理をまとめる。本アプリの機能は、単一教科の箱ではなく、
> 探究サイクルの中で他教科と往復することを前提に設計される。

---

## 🎯 設計原則

1. **社会科を主軸に据える**: 本単元の中心の問い・探究の核・評価の軸は社会科。他教科は**貢献の仕方を選ぶ**立ち位置
2. **押しつけない**: 教員が教科横断を採用しない選択も尊重。単元設計 UI で ON/OFF 可能
3. **探究のサイクルに沿う**: 調べる → 立場を可視化 → 書く → 表現する → 他者に伝える、の流れで各教科が役割を持つ
4. **「他者性」「立ち止まり」「AI に出てこない誰か」は全教科共通の軸**: どの教科も、この視座を失わない
5. **作品は必ず「誰かへ向けて」作る**: 表現は自己表出で終わらず、クラス・他学年・家族・地域への伝達を意図

## 🧭 学習指導要領との関係(2017 告示・社会編ほか)

### 社会科(主軸)
本アプリが直接的に支援する資質・能力(抜粋)
- 多面的・多角的に考える力(複数立場の比較)
- 事実と意見を区別し、根拠を持って論じる力(出典明示)
- 社会に主体的に関わり、公共的判断を仮り組む力(立場選択 → 振り返り → 再選択)
- **批判的に情報を扱う力**(AI 出力の吟味、「AI に出てこないのは誰?」)

### 国語
- 書く活動(B 領域): 立場のナラティブ、振り返り記述、動画ナレーション台本
- 話す・聞く(A 領域): クラス内の意見交流、少数派さがしの討議
- 読む(C 領域): 資料の多角的読解、出典を明示した引用

### 図画工作
- A 表現: 「声が聞こえていない人を絵にする」 Gemini 画像 + 児童の手入れ
- B 鑑賞: 友だちの「立場の絵」を見て感じたことを言葉にする
- 形や色の選択が**誰に何を伝えたいか**と結びつく

### 音楽
- A 表現: 立場の「気持ち」を 30 秒程度の簡易メロディと BGM で表す
- B 鑑賞: 他の児童が作った「立場のテーマ」を聞き、印象を言葉にする
- Web Audio API / Tone.js で**音符を選ばない**簡易作曲(音楽的素養に依存しない)

### 総合的な学習の時間(探究)
- 単元全体の「探究の時間」として位置付け可能
- 地域調査・インタビュー・他校交流との接続

### プログラミング教育
- 「つくってみようモード」で立場当てクイズやアンケートアプリを児童が作る
- 論理構成(条件分岐 = 立場に応じた応答)という社会的思考の形式化

## 🎨 図工との接続:具体設計

### 活動の例
- **「地図にない声を絵にしよう」**
  立場マップに出てこなかった存在(動物、将来世代、引っ越してしまった人)を絵にする。
  Gemini 画像生成 + 児童の手直し(色・タイトル・一言)。
- **立場のコラージュ**
  複数の立場の絵・写真・記号を1枚の紙(デジタルキャンバス)に配置し、関係を見せる。

### アプリでの支援
- 画面 8 の画像プロンプトコーチが、**「声が聞こえていない人の絵」**をテーマ化
- Gemini が出した複数候補から児童が選び、**色・タイトル・注記を児童が書き足す**(AI は叩き台、表現は児童)
- 鑑賞: しらべもの広場で絵の横に「どんな気持ち?」スタンプで感想を交換

### 評価観点
- 形・色の選択が「伝えたい相手・気持ち」と結びつくか
- 他の児童の作品を見て、自分にない視点を受け取れるか

## 🎵 音楽との接続:具体設計

### 活動の例
- **「この立場のテーマ BGM を30秒で」**
  選んだ立場の「気持ち」を表すメロディ+和音+リズムを選ぶ(楽譜不要)。
- **「少数派のための小さな音」**
  大きな音(多数派)の中に紛れる小さな音(少数派の声)を残す構成。

### アプリでの支援(Phase 3)
- 画面 21(動画つくろう)の BGM 選択と連動
- `lib/music/` が mood ラベル(やさしい・しずか・さびしい・たしかな・軽やか)から
  Tone.js で短いメロディを合成。児童は**mood の組み合わせを選ぶ**
- 作曲そのものは AI 任せでよい(音楽的素養で差がつかないことを重視)
- ただし「なぜこの mood を選んだか」は児童が書く(国語)

### 評価観点
- 立場の感情を**音の性質**で言葉以外に表現しようとしたか
- 自分と違う気持ちの音を聴くときの受け取り方

---

## ✍️ 国語との接続:具体設計

### 活動の例
- **「わたしの知らなかった誰かの声」作文**
  `MissingVoiceHypothesis` で自分が書いた仮説を土台に、その立場から見た風景を 200〜400 字で書く
- **振り返りを論述に**
  単元の事後振り返り 3〜5 回を、整理して 600〜1000 字の論述に組み直す(upper 向け)

### アプリでの支援
- 動画つくろう(21)のナレーション台本は**国語の書く指導**として扱える
- Claude は「書く前の問い」のコーチだけ提供(書く本体は児童)
- 立ち止まりの言葉の検出は、**推敲の手がかり**として児童に見せる

### 評価観点
- 出典を示して書けるか
- 自分と違う立場の人に読まれる前提で書けるか
- 断定を避け、仮説として書けるか(「〜かもしれない」「〜のように思う」)

---

## 🧪 総合(探究)との接続

### 単元全体を「総合」として位置付ける
- 実習Ⅲが想定する「地域社会と接点のある題材」の 10〜15 時間
- 社会科の学習内容と結びつけながら、探究のサイクルを自覚する

### 地域取材の拡張
- インタビュー: 取材計画 → 実行 → 記録 → ナレッジ登録(出典)→ 対話 → 再訪問
- 他校交流: しらべもの広場のクラス跨ぎ機能(Phase 4 で検討)

---

## 💻 プログラミング教育との接続

### つくってみようモード(画面 10)
- 単元で出てきた立場の**立場当てクイズアプリ**を児童が作る
- クラスで動かして遊ぶ → みんなが立場の多様性を体感
- プログラミングが「社会的なもの」を扱う体験に

### 例題
- 「あなたはどの立場? 5つの質問でわかるアプリ」(条件分岐の自然な導入)
- 「立場マップの投票アプリ」(データ収集と可視化)
- 「少数派の声をあつめる意見箱」(入力と保存、ただしモデレーションは基盤機能)

---

## 🧭 教科横断の**やらないこと**

- **全教科を同じ重みで扱う**ことはしない。教員が選べる
- **教科のねらいを曖昧にする**ことはしない。各教科の評価観点は明示
- **他教科の時間を強制する**ことはしない。「接続可能な場面の提示」で止める
- 教科横断を「同時進行」にしない(社会の時間に音楽もする、ではなく、社会で出た立場を後日の音楽で表現する)

## 📊 教員が使うマッピングの UI

単元設計画面(13)で、各 hour に教科横断タグを付けられる:

```
h11  表現:動画/音楽/作文                      [none]
      📚 社会  ✍️ 国語  🎨 図工  🎵 音楽        🎯 表現
```

`Unit.crossCurricular` に JSON で保存、各 `UnitHour` にもタグ可。
